# Token System - How design.json Controls All Tokens

## Three-Layer Token Architecture

Your website now uses the same three-layer token system as the CMS:

### 1. Foundation Layer (Blimpify UI)
Defined in `@blimpify-im/ui` package:
- `--foundation-layout-sm-content`, `--foundation-layout-md-content`, etc.
- `--foundation-section-spacing-sm`, `--foundation-section-spacing-lg`, etc.
- `--foundation-container-spacing-md`, etc.
- `--foundation-navbar-spacing-md`, `--foundation-navbar-spacing-lg`, etc.
- `--foundation-radius-full-xl`, `--foundation-radius-md-lg`, etc.

### 2. Selected Layer (design.json → themeEngine.ts)
Generated by `buildCssVars()` from your `design.json`:
```json
{
  "globalStyles": {
    "layoutContent": "sm",      // → --selected-layout-scale-content: var(--foundation-layout-sm-content)
    "layoutMedia": "sm",        // → --selected-layout-scale-media: var(--foundation-layout-sm-media)
    "sectionSpacing": "lg",     // → --selected-section-spacing: var(--foundation-section-spacing-lg)
    "containerSpacing": "md",   // → --selected-container-spacing: var(--foundation-container-spacing-md)
    "navbarSpacing": "lg",      // → --selected-navbar-spacing: var(--foundation-navbar-spacing-lg)
    "radius": "full"            // → --selected-radius-scale-xl: var(--foundation-radius-full-xl)
  }
}
```

### 3. Semantic Layer (themeEngine.ts → Components)
Semantic tokens that components actually use:
```css
/* Width tokens */
--width-content: clamp(18rem, 88vw, var(--selected-layout-scale-content));
--width-navbar: var(--width-content);
--width-container: var(--width-content);
--width-media: clamp(18rem, 92vw, var(--selected-layout-scale-media));

/* Spacing tokens */
--space-section: var(--selected-section-spacing);
--space-container: var(--selected-container-spacing);
--space-navbar: var(--selected-navbar-spacing);

/* Radius tokens */
--radius-navbar-pill: var(--selected-radius-scale-xl);
```

## How Components Use Tokens

### StaticNavbar
```css
.navbar-pill__container {
  max-width: var(--width-navbar);      /* ← from layoutContent: "sm" */
  margin: var(--space-navbar) auto;    /* ← from navbarSpacing: "lg" */
  padding: var(--space-navbar);        /* ← from navbarSpacing: "lg" */
  border-radius: var(--radius-navbar-pill); /* ← from radius: "full" */
}
```

### Section Component (from Blimpify UI)
```css
.section {
  padding-top: var(--space-section);    /* ← from sectionSpacing: "lg" */
  padding-bottom: var(--space-section); /* ← from sectionSpacing: "lg" */
}
```

### Container Component (from Blimpify UI)
```tsx
<Container useNavbarWidth>  {/* uses --width-navbar */}
<Container useMediaWidth>   {/* uses --width-media */}
<Container>                 {/* uses --width-content (default) */}
```

## Controlling Tokens from design.json

Edit `/web/public/design/design.json`:

```json
{
  "globalStyles": {
    "layoutContent": "sm",      // content width: sm | md | lg | xl
    "layoutMedia": "sm",        // media width: sm | md | lg | xl
    "sectionSpacing": "lg",     // vertical spacing: 2xs | xs | sm | md | lg | xl | 2xl
    "containerSpacing": "md",   // container padding: xs | sm | md | lg | xl | 2xl
    "navbarSpacing": "lg",      // navbar padding/margin: xs | sm | md | lg | xl | 2xl
    "radius": "full",           // border radius: none | sm | md | lg | xl | full
    "formWidth": "sm"           // form width: xs | sm | md | lg
  }
}
```

Changes to design.json are immediately reflected on page reload (no rebuild needed).

## Token Flow Example

When you set `"navbarSpacing": "lg"` in design.json:

1. **Foundation**: `--foundation-navbar-spacing-lg: 24px` (defined in Blimpify UI)
2. **Selected**: `--selected-navbar-spacing: var(--foundation-navbar-spacing-lg)` (generated by themeEngine)
3. **Semantic**: `--space-navbar: var(--selected-navbar-spacing)` (generated by themeEngine)
4. **Component**: `.navbar-pill__container { margin: var(--space-navbar) }` (uses semantic token)

Result: Navbar gets 24px margin!
